## Overview
A Button is used consistently but sparingly in a web page. The nature of the action performed by the button is supported through variants.

Sometimes, a button is complemented by an icon. Icon only buttons are also common usage.

A counter in the trailing position is a common pattern at Github. Buttons can denote state by changing icon color or text.

Button also act as a bedrock for the DropdownMenu. Buttons can be grouped also.

Buttons need to visually and non-visually support different user actions and states like hover, disabled, focused and active.

## Accessible Buttons

The Button component has to follow the below criteria to be accessible

- Sufficient color contrast during the various states(focus/hover/pressed) of the button in each variant. 
- Correct button text even in icon only buttons
- Make `aria-hidden` for icons.
- Treat link as a link and button as button.

## Button sizes

Button components will be available in two sizes.

Standard(32px) and Small(28px)
